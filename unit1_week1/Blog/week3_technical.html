Arrays vs. Hashes

What is an array?

An array is basically a collection or list of things. The things can be numbers or strings or booleans. The things in an array are stored in a specific order.

What is a Hash?

A hash is similar to an array in that it is also a collection--but in this case a hash has a set of key and value pairs. You can think of a hash as a dictionary. It matches a word (or another object) to a defnition (or value). Unlike arrays, the things in a hash can be stored in any order.

cheese = cheddar

But let's say you wanted to store more than one kind of cheese...you can do this with an array:

cheese = ["cheddar", "swiss", "gouda", "muenster", "brie"]

Now you can access and work with any of the cheeses stored in the array.

What is a hash good for?

A hash is good for organizing data also. Using the cheese example, let's say we wanted to take inventory of how many of each kind of cheese we have left in our cheese shop:

cheese = {"cheddar" => 5, "swiss" => 2, "gouda" => 2, "muenster" => 3, "brie" => 4}

Now you can work with this hash to update or modify your inventory of cheeses.
